# Local Development Guide

This guide will help you set up your local environment to work on **The One (eluno.org)**.

## ðŸ“‹ Prerequisites

Before starting, make sure you have installed:

1. **Node.js**: Version 20.x or higher.
   - Verify: `node -v`
2. **Git**: For version control.
3. **Code Editor**: We recommend VS Code or Cursor.

## ðŸš€ Installation and Setup

1. **Clone the repository:**
   ```bash
   git clone https://github.com/chuchurex/eluno.org.git
   cd eluno.org
   ```

2. **Install dependencies:**
   ```bash
   npm install
   ```

3. **Configure environment variables:**
   - Copy the example file:
     ```bash
     cp .env.example .env
     ```
   - Open `.env` and configure basic variables (the project can run locally without external credentials, but some build/deploy functions will require them).

## ðŸ’» Running Locally

To start the development server with hot reload and SASS compilation:

```bash
npm run dev
```

This command:
- Compiles `src/scss/main.scss` to CSS.
- Watches for changes in SCSS files.
- Starts a local server at `http://127.0.0.1:3002`.
- Opens the browser automatically (depending on your config).

## ðŸ›  Available Commands

| Command | Description |
|---------|-------------|
| `npm run dev` | Start development environment (SASS watch + Live Server). |
| `npm run build` | Generate production version in `dist/` folder. Compiles HTML and CSS. |
| `npm run sass:watch` | Only watch and compile SASS changes. |
| `npm run serve` | Serve the `dist/` folder statically (useful for testing final build). |
| `npm run translate` | Run translation scripts (requires API Keys). |
| `npm run build:pdf` | Generate chapter PDFs (uses Puppeteer). |

## ðŸ“‚ Project Structure

- **`src/`**: Source code.
  - `scss/`: Styles (SASS).
  - `fonts/`: Local font files.
- **`i18n/`**: Content.
  - `en/`, `es/`, `pt/`: JSON files with chapter text and UI.
- **`scripts/`**: Build logic and Node.js tools.
- **`dist/`**: Output folder (what gets deployed). Generated by `npm run build`.

## ðŸ§ª Typical Workflow

1. **Edit Content:** Modify JSON files in `i18n/`.
2. **Edit Styles:** Modify SCSS files in `src/scss/`.
3. **Preview:** See changes in real-time with `npm run dev`.
4. **Test Build:** Run `npm run build` and review `dist/` if you have questions about final generation.
